## Introduction and Purpose

This application will be a elegant task manager that will allow users to keep track of their tasks througout their own chosen periods of time.  It will have the purpose of allowing users to keep track of their tasks on the desktop environment, and it will overall encourage more user productivity and satisfaction.

## Overall Description

### Context:

This application/system will be used in the context that users will need to keep track of tasks throughout their busy schedules.  Many of these users will want a system that is simple, elegant, and easy to navigate.

### User Characteristics:

Many of the users of this application will either be young and working-age, or in their later working years (middle-aged to retired).  Many of these users will have busy corporate jobs, and will need organization in order to effectively categorize and complete their many tasks.  Other than this core group of users, everyday usage is also accounted for in this app (such as dealing with chores and such).  Many of these people will be experienced with technology, but some will be more on the novice side.

### Assumptions:

1. This product, at its base level, must provide users with the ability to add tasks and information to their to-do lists in a highly customizable environment.  
2. Users must have a lot of freedom to choose how their to-do lists look, but they are also given the choice to automatically add suggested tasks from the system itself.
3. Users must be able to complete tasks in their to-do list so that they can focus on other tasks.

## Functional Requirements

1. Core Function: Users can add tasks and information to the to-do list
2. Tasks can be automatically added to the user's schedule
3. Users can delete tasks
4. Users can set an ajustable amount of days for reminders
5. Users can create and manage multiple to-do lists
6. Tasks can be integrated around existing events and/or classes during the day
7. Users can change the app from light mode to dark mode
8. Users can categorize/tag reminders/todos and label priority levels
9. Users can change the color or background image of their planner
10. Users can restrict the kinds of tasks that they can complete in a certain interval for organization
11. This applicagtion will have a functional API
12. Users will recieve a list of suggested tasks that they can pick from for their task managers
13. Users will recieve desktop notifications for their upcoming tasks
14. Users will be able to merge their task lists together

## Non-Functional Requirements

1. The system should not lag or studder when a user interacts with tasks (adding & removing)
2. uptime should be atleast 90%
3. tasks should not be lost or assigned to the wrong categories.
4. any task past its completion date should not stay in the list. 

## Use Cases or User Interactions

1. users will remove and add tasks to the to-do lists.
2. a data base will store to-do lists and their tasks.
3. users will create categories in their to-do lists.
4. users will modify existing tasks.
5. users will block out time during each day they can get tasks done
5. the the program will schedule tasks during the week using the users blocked out time

## Assumptions and Constraints

1. assume the user will have basic computer literacy (know how to use technology).
2. assume that the user will have a stable electricity to use the task manager (desktop or phone)
3. assume user have internet access.
4. contraint of using only java, python, react.
5. constraint of the app is only used online on a website.
6. constraint of the app needs to be done within the next three months.
